Iteration 3 Tested 14 Jan 2021
Key findings of iteration 3

Information grouping created from the card sort workshops that were tested in this iteration:

Prepare for the induction appointment:

  • Appointment date and time
  • Information to identify the service user
  • Contact details for the service user
  • Personal circumstances (from pre-sentence report if applicable)
  • Case information (High level information of offence, sentence/order and risk level)

Prepare for the second appointment (Initial Sentence Plan):

  • Appointment date and time
  • Background from last appointment
  • More detail on the offence
  • More detail of risk level and past risk assessments
  • Previous order details
Iteration 2 Tested 3 Dec 2020
Key findings of iteration 2

User stories created from the co-design workshops that were tested in this iteration:

As a Probation Practitioner, in order to prepare for an appointment I need to...

  • ...see their previous offences
  • ...see their risk score
  • ...see their past appointments
  • ...see their how much time is left on their current order / RAR days
  • ...see my past notes

We found that reusing the 'Prepare a case for sentence' prototype was making it difficult to know how users needed the information to be structured.

This preventing us from getting to the core user needs, so we decided before iterating further that we would run card sorting workshops on 11 December and 7 January with Probation Practitioners, asking them to group information and prioritise when they would need the information.

The card sort workshops allowed us to create information groupings that would inform the Information Architecture (see Iteration 3 notes).

Iteration 1 Tested 13 Nov 2020
Key findings of iteration 1

We received a lot of positive comments about the GOV.UK style and how it was better than the current system.

This preventing us from getting to the core user needs, so we decided before iterating further that we would run co-design workshops on 24-25 November with Probation Practitioners, asking the question "How might we make the start of the probation journey better?"

The co-design workshops allowed us to create user stories for the start of the probation journey (see Iteration 2 notes).
